# 简历项目内容

## 项目名称

AI Agentic 演示稿生成系统

## 一句话描述

基于 LLM Agent 的智能演示稿生成产品，用户输入原始内容，系统自动完成内容理解、结构规划、交互澄清、改写润色，产出可直接发布的网页演示稿。

---

## STAR 写法（简历正文）

**S：** 用户有讲稿、笔记、大纲等原始内容，但缺乏将内容转化为结构清晰演示稿的能力。现有工具要么只做格式渲染（无内容理解），要么是重型编辑器（学习成本高），市场缺少"懂内容"的轻量智能生成方案。

**T：** 从 0 到 1 主导产品定义与落地，目标是构建一个 Agentic 演示稿生成系统，让用户只需输入原始内容，即可获得可直接分享的网页演示稿。

**A：**

- 定义核心产品问题：将产品价值锚定为"内容→演示结构的智能转化"，而非渲染工具，以"用户生成后改动量"作为北极星指标
- 设计 6 阶段 Agentic Pipeline：结构分析 → 主动澄清 → LLM 规划大纲 → 自检纠偏 → 演示语言改写 → 确定性渲染
- 制定"LLM 主脑 + 系统轨道"架构原则：LLM 负责内容理解与语义决策，系统负责兜底、缓存、状态机与质量约束
- 设计 Human-in-the-Loop 机制：基于 LLM 输出的 planning_confidence 动态触发澄清，最多问 1-2 个关键问题
- 建立质量评估体系：定义 7 个自动评分维度和 5 个 Agent 评估指标，驱动迭代优化
- 借助 AI 编程工具协作完成全栈工程实现，亲身验证 LLM 能力边界

**R：**

- 完成端到端 Agentic 产品 MVP，覆盖从内容输入到演示稿导出的完整链路
- 实现 LLM 驱动规划 + 规则兜底的混合可靠架构，LLM 不可用时自动降级，保证 100% 可用性
- 落地 Studio 在线编辑器，支持实时预览、主题切换、大纲确认的完整用户体验
- 形成可复用的 Agent 产品方法论：LLM 与规则的边界划分、最小化澄清设计、Agent 质量评估框架
